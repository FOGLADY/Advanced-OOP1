Unit 1: Introduction and Recap of Object-Oriented Programming

Unit Overview

This unit revisited the four core principles of object-oriented programming: abstraction, encapsulation, inheritance and polymorphism. It also explored the role of classes, objects, constructors and access control in secure and maintainable software design.

Artefact

I created a small Python AI model hierarchy using an abstract base class and two specialised subclasses. The artefact demonstrates private attributes, controlled access through properties, inheritance and different implementations of the same predict() method.

What I Learned

The activity helped me understand how a shared abstraction can provide a consistent interface while allowing subclasses to implement different behaviours. Encapsulation also demonstrated how object state can be protected from uncontrolled access.

Evidence

  • Abstract AIModel base class
  • TextClassifier and ImageClassifier subclasses
  • Successful execution in the PyCharm console

Unit 2: SOLID Principles of Object-Oriented Design

Unit Overview

This unit introduced the five SOLID principles: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation and Dependency Inversion.

Artefact

I first created an over-responsible AIApplication class that performed model selection, prediction, result storage and console output. The code was then refactored into focused interfaces and classes following SOLID principles.

What I Learned

The comparison demonstrated that separating responsibilities makes code easier to extend, test and maintain. A new predictor could be introduced without modifying the existing prediction service, which demonstrated the Open/Closed Principle.

Evidence

  • Code before SOLID refactoring
  • Separate Predictor, ResultRepository and Reporter abstractions
  • Text, image and audio predictor implementations
  • Successful execution of the refactored application
